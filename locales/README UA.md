🌍 Локалізація інтерфейсу
Ця папка містить файли перекладів для RimWorld Translator Grabber.

📁 Структура файлів
Кожна мова зберігається в окремому файлі з простою назвою:
locales/
├── ru.json      # Русский
├── en.json      # English
├── ua.json      # Українська
├── ja.json      # 日本語
└── README.md    # Цей файл

🚀 Як додати нову мову (5 хвилин)
**Крок 1: Створити файл перекладу**
Скопіюйте існуючий файл (наприклад, `en.json`) і перейменуйте його на `{код_мови}.json`:
# Приклад для німецької мови:
copy locales\en.json locales\de.json

Популярні коди мов:
- `de` — Deutsch (німецька)
- `fr` — Français (французька)
- `es` — Español (іспанська)
- `it` — Italiano (італійська)
- `pl` — Polski (польська)
- `zh` — 中文 (китайська)
- `ko` — 한국어 (корейська)

**Крок 2: Перекласти значення**
Відкрийте файл і перекладіть тільки значення (права частина). НЕ змінюйте ключі!
{
  "de": {
    "app_title": "RimWorld Translator Grabber",  ← перекладено
    "menu_file": "Datei",                        ← перекладено
    "menu_exit": "Beenden",                      ← перекладено
    ...
  }
}
❌ НЕ МОЖНА:
{
  "menu_file": "Datei",  ← ключ змінено — ПОГАНО!
}
✅ МОЖНА:
{
  "menu_file": "Datei"  ← ключ збережено, значення перекладено — ДОБРЕ!
}

**Крок 3: Перевірити файл**
python -c "import json; json.load(open('locales/de.json',encoding='utf-8')); print('✅ Файл валідний!')"
Або розширена перевірка:
python scripts/validate_locales.py locales/de.json

**Крок 4: Протестувати**
- Запустіть програму
- Відкрийте меню 🌐 Мова інтерфейсу
- Виберіть нову мову
- Перевірте, чи всі тексти перекладені

**Крок 5: Надіслати PR**
Створіть Pull Request із файлом перекладу.

🔧 Автоматичне встановлення
Нова мова з'являється автоматично за наявності файлу `locales/{код}.json`.
Більше нічого реєструвати не потрібно!

📝 Правила перекладу
**Форматування**
✅ Використовуйте кодування UTF-8
✅ Зберігайте емодзі (📦, ✅, 🌐 тощо)
✅ Зберігайте пробіли в кінці рядків (якщо вони є в оригіналі)

**Спеціальні символи**
- `\n` — новий рядок
- `\"` — лапка всередині рядка
- `\\` — зворотній слеш

**Статуси перекладу**
На початку файлу можна додати коментар:
{
  "_meta": {
    "language": "de",
    "translator": "YourName",
    "last_updated": "2026-04-05",
    "completion": "100%"
  },
  "de": {
    ...
  }
}

🐛 Проблеми та рішення
**Переклад не завантажується**
- Причина: Файл не валідний JSON
- Рішення: Перевірте файл через `python -m json.tool locales/de.json`

**Деякі рядки не перекладені**
- Причина: Відсутні ключі
- Рішення: Порівняйте з `en.json` і додайте відсутні

**Текст обрізається**
- Причина: Занадто довгий переклад
- Рішення: Скоротіть текст або використайте скорочення

💡 Поради
- Використовуйте оригінальний файл як шаблон — не починайте з нуля
- Контекст важливий — ключі показують, де використовується рядок:
  - `menu.*` — пункти меню
  - `tab.*` — назви вкладок
  - `editor.*` — редактор перекладів
  - `dialog.*` — діалогові вікна
  - `status.*` — рядок стану
- Тестуйте часто — запускайте програму після кожних 50-100 рядків
- Зберігайте емодзі — вони частина інтерфейсу
